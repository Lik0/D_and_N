# Квест-карта

Статический сайт для GitHub Pages с интерактивной картой на Leaflet и прогрессом по городам Калининград и Гродно.

## Запуск на GitHub Pages
1. Откройте **Settings → Pages** вашего репозитория.
2. В разделе **Source** выберите **Deploy from a branch**.
3. Установите ветку **main** и папку **/(root)**.
4. Сохраните — GitHub Pages соберёт сайт автоматически.

## Локальное тестирование
Для корректной работы `fetch` статических файлов нужно запускать простой сервер:
```bash
python -m http.server
```
Откройте в браузере [http://localhost:8000](http://localhost:8000).

## Редактирование точек
Данные лежат в `data/places.json`.
- В секции `cities` можно обновить центр и зум для города.
- В массиве `places` каждая точка описывается полями:
  - `id` — уникальный идентификатор (например, `k01`, `g01`).
  - `city` — `kaliningrad` или `grodno`.
  - `order` — порядок открытия (начиная с 1).
  - `lat`, `lng` — координаты.
  - `title` — заголовок точки.
  - `story` — воспоминание, показывается после правильного ответа или при повторном открытии.
  - `quiz` — объект с полями `question`, `options` (4 варианта) и `correctIndex` (номер правильного варианта с 0).

Добавьте новые объекты в массив, соблюдая уникальность `id` и последовательность `order` внутри города. После изменения файла перезапустите страницу.
